### 1Ô∏è‚É£ üîê **AuthSystem** ‚Äî Secure Authentication Backend ‚úÖ COMPLETE

**Location:** `Projects/AuthSystem`

**Tech Stack:**
- Node.js, Express.js, MongoDB, Mongoose
- JWT (access + refresh tokens)
- Zod validation
- bcrypt password hashing
- Nodemailer email service
- HTTP-only secure cookies

**Key Features:**
- User registration with email verification
- Login with JWT-based session management
- Access & Refresh token strategy
- Token expiry and renewal flows
- Centralized error handling
- Request validation middleware
- Production-ready folder structure

**What You'll Learn:**
- Complete auth lifecycle (signup ‚Üí email verify ‚Üí login ‚Üí token refresh ‚Üí logout)
- JWT best practices
- Secure cookie management
- Password hashing strategies
- Email service integration

**To Run:**
```bash
cd Projects/AuthSystem
npm install
cp .env.example .env
# Add: MONGO_URL, JWT_SECRET, JWT_REFRESH_SECRET, NODEMAILER_PASS, BASE_URL
npm run dev
```

**API Endpoints Example:**
- `POST /api/auth/signup` - Register user
- `POST /api/auth/login` - Login user
- `POST /api/auth/verify-email` - Verify email token
- `POST /api/auth/refresh-token` - Refresh access token
- `GET /api/auth/me` - Get current user (protected)

## üöÄ Quick Start (General Pattern)

Each project follows a similar setup pattern:

```bash

cd Projects

# Navigate to specific project

# Setup environment

# Edit .env with your credentials

# Run development server
npm run dev
```
